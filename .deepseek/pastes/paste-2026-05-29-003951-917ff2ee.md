   四、核心架构问题                                                          
                                                                             
   ```                                                                       
     声明的 5 层架构:  Utils → Storage → Network → Dispatch → UI             
     实际的调用关系:  app.zig ──→ stream_client.zig ──→ DeepSeek API         
                         ↓                                                   
                     (跳过了 Storage、Cache、Dispatch、Skills、Agent         
   等全部中间层)                                                             
   ```                                                                       
                                                                             
   最严重的问题：app.zig 与声明的架构严重不符。约 15,000                     
   行精心设计的子系统代码（缓存、调度、存储、技能、代理、沙箱、Provider      
   管理等）处于"已实现但未接线"状态。主应用只使用了最基础的流式 API 调用 +   
   原始 ANSI 渲染，像一个与豪华后端脱节的极简前端。
